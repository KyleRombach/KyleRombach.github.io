Enumerable#map
2/1/2016

Let's talk first what about what Enumerables are. They are methods that we use to change and sort our Arrays and Hashes. They allow you to change these in a more streamlined fashion. They must use an each iterator becuase they need to work throughout the entire collector.

We are gonna learn about what the Enumerable map does.Map allows use to take an Array or Hash and then make a new one based on what we set the block to. Since all we are doing is editing the elements the array length should return the same. If you use the map! method it will do the the same thing but instead of just proccessing the original and printing a new one, it will actually edit the original so the array or hash is now set to the new values. 

          [10, 12, 14, 16,  18, 20].map { |x| x / 2 }

          names = ["Tom", "Sylvester", "Garfield", "Simba"]
          p names

          names.map! { |names| names + " is a famous animated cat" }
          p names

          [5, 6, 7, 8, 9, 10]

          ["Tom", "Sylvester", "Garfield", "Simba"]
          ["Tom", "Sylvester", "Garfield", "Simba"]

          ["Tom is a famous animated cat", "Sylvester is a famous animated cat", "Garfield is a famous animated cat", "Simba is a famous animated cat"]
          ["Tom is a famous animated cat", "Sylvester is a famous animated cat", "Garfield is a famous animated cat", "Simba is a famous animated cat"]

So you can see here from the examples that when I mapped the array of numbers and set the block to divide them by two, it will return a new array with those results. Then for the names array you can see how I was able to edit the strings and also made a permanent change to the array. When I called the array after the map! proccess it only gave me the results of the new array. These are just some basic examples of how to use the map method and you can learn even more ways to implement it with different blocks.
