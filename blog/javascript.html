JavaScript
2/16/2016

In Ruby you use Hashes as one way to store information. In JavaScript they have something similar but a little different called an object. With a Hash you set a key and give it a value. It would look like this.

hash = { key1: 'one', key2: 'two', key3: 'three'}

In JavaScript the object will look similar but with a slight difference. You have to add var in front of the object you are creating.

var hash = { key1: 'one', key2: 'two', key3: 'three'}

As you can see they look pretty much the same minus adding the variable in the front. Now how you access the information stored is pretty different. First lets see how you would access it in Ruby. Next I'll show how you access it in JavaScript.

hash[:key1] --> 'one'

hash.key1 --> 'one'

As you can see they are similar with some minor differences. You call on the hash to get the value of the key. For those of you who don't know also with the :key1 syntax, that means I made the key a symbol. You have the option to have a key be a string as well (i.e. 'key1'). These both work but symbols will load faster. For JavaScript you use the dot method to access the value of the object.You can also store Arrays as a value for either a Hash or Object but that is something that we will talk about later. This is just a simple difference between Hashes and Objects are there are some other factors to take into account as well.
